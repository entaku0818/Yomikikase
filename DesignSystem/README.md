# ナレーター デザインシステム

VoiceYourText（ナレーター）の UI を「色・操作・余白」の 1 つのルールに統一するためのデザインシステム。
本フォルダは、Figma/キャンバスのビジュアル案に対応する **実装ハンドオフ** と、その運用ルールを格納する。

- バージョン: v1.0（2026-06-07）
- 対象: iOS（SwiftUI / TCA）
- 確定ブランドカラー: **インディゴ `#4B47E0`**

---

## 1. なぜやるか（背景）

`AppTheme.swift` には「アクセントは Indigo で統一」「グラデーション不使用」と明記されているのに、本体 UI では守られていない。現状の 3 つの矛盾:

1. **ホームのアイコンが 7 色** に分かれている（青・赤・紫・緑・茶・グレー・水色）
2. **再生ボタンの色が画面ごとに違う**（テキスト=青 / PDF=オレンジ / ミニプレイヤー=黒）
3. **プレイヤーの操作系がテキストと PDF で全く別物**。ユーザーは同じ「再生」を画面ごとに学び直す必要がある

→ ブランドの一貫性が崩れ、学習コストが高い。

## 2. ゴール（完了の定義）

- [ ] **色**: 意味を持つ UI（操作・誘導）はすべてアクセント 1 色。機能ごとの色分けを全廃
- [ ] **操作**: 再生・一時停止・スキップは、どのコンテンツでも同じ部品・同じ並び（共通 PlayerBar）
- [ ] **基盤**: 色・余白・角丸・影・タイポが少数のトークンに収束し、`AppTheme` が唯一の出どころ
- [ ] **ネイティブ準拠**: SF Symbols・大タイトル・グループ化リスト・システムカラーを尊重し、ライト/ダーク両対応
- [ ] ライト/ダーク両方で全画面のコントラスト確認（最小フォント 12・ヒット領域 44×44pt）

## 3. 統一の 3 原則（暗記用）

| 原則 | 内容 |
|---|---|
| **1 色ルール** | 意味を持つ UI（操作・誘導）はすべてアクセント 1 色。機能ごとの色分けは廃止 |
| **1 操作系ルール** | 再生・一時停止・スキップは、どの画面でも同じ部品・同じ並び |
| **ネイティブ準拠** | SF Symbols・大タイトル・グループ化リスト・システムカラーを尊重し、ライト/ダーク両対応 |

**アクセントを使う場所** = 再生ボタン・全アイコン・CTA・タブのアクティブ・進捗バー・選択中のフィルタ。
それ以外（文字・背景・区切り線）はすべてニュートラル。**グラデーション禁止。**

## 4. スコープと進め方

実装はチェックリスト（→ [`design-unification-spec.md`](./design-unification-spec.md) §07）を GitHub issue に分解して進める。

| フェーズ | 内容 | 状態 |
|---|---|---|
| **Foundation** | `AccentIndigo` カラー登録 / `AppTheme` トークン拡張 / ホームの 7 色アイコン統一 | 着手中 |
| Unified Player | 共通 `PlayerBar` 新設、PDF/テキスト/ミニプレイヤーの再生 UI 置換 | issue 化 |
| Per-Screen | マイファイル / 設定 / 課金 / オンボーディングの統一 | issue 化 |
| QA | ライト/ダーク・コントラスト・ヒット領域の最終確認 | issue 化 |

## 6. 運用ルール

- 色・余白・角丸の **新しい値を増やさない**。必要なら `tokens.md` を更新してから使う。
- `Color.indigo` / `Color.orange` などの **直書きを禁止**。すべて `AppTheme` 経由。
- 仕様変更は本フォルダ（特に `tokens.md`）を更新してからコードに反映する。
- ビジュアルの最終確認は `assets/source-html/` のキャンバスを基準にする。
